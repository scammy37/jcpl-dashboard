Options A, B, and D work from any browser with no setup. Only Option C (the script) requires the repo cloned locally with Python installed. Follow these steps once per machine.

Step 1 — Install Git (if not already installed)
1
Go to git-scm.com/download/win and download Git for Windows
2
Run the installer — default options are fine
3
Verify: open PowerShell and run git --version — should print a version number

1
Go to python.org/downloads and download the latest Python 3.x installer
2
Run the installer — check "Add Python to PATH" before clicking Install
3
Verify: open PowerShell and run python --version — should print a version number
Step 3 — Clone the repo
1
Open PowerShell and navigate to where you want the folder:
cd C:\Claude (or wherever you prefer)
2
Clone the repo:
git clone https://github.com/scammy37/jcpl-dashboard.git
3
Enter the folder:
cd jcpl-dashboard
Step 4 — Configure Git identity (first time only)
1
Run:
git config --global user.name "Michael Zagame"
git config --global user.email "michael.zagame@gmail.com"
2
When you first push, GitHub will ask you to log in — use your GitHub credentials or a personal access token
Step 5 — Test it
1
From inside the jcpl-dashboard folder, run:
python serve.py
2
Open http://localhost:8080 — the dashboard should load with your live data
3
You're ready to use Option C
💡 Work PC note: some corporate networks block GitHub or require a VPN. If git clone fails, try connecting via VPN first or use Options A, B, or D which only need a browser.

🤖 Option A — Auto-Fetch Automatic

A GitHub Actions workflow runs automatically from the 7th–12th of each month. It logs in to firstenergycorp.com using your saved browser cookies, downloads the latest bill PDF, parses it, and adds the entry to data.json — no action needed from you.

You'll receive an email at michael.zagame@gmail.com with one of these results:
✓
New bill added — entry was parsed and pushed to the dashboard automatically
!
Session cookies expired — log in to firstenergycorp.com in Chrome, open DevTools → Network, right-click any request → Copy as cURL, extract the Cookie: header, and update the JCPL_COOKIES secret at github.com/scammy37/jcpl-dashboard/settings/secrets/actions
!
Any other failure — check the Actions run log and fall back to Option B, C, or D below
💡 You can also trigger the workflow manually: go to github.com/scammy37/jcpl-dashboard/actions → Fetch JCP&L Bill → Run workflow.

💬 Option B — Use Claude Any device

Works on your phone, tablet, or any computer. Go to claude.ai, attach your JCP&L bill PDF, and paste this prompt:

Extract the billing data from this JCP&L electricity bill and return a single JSON object in this exact format — no markdown, no explanation: { "label": "Mon YY", "period": "Mon DD–Mon DD, YYYY", "days": 30, "kwh": 1414, "cost": 282.62, "temp": 72, "rate": "Standard", "onPeak": null, "offPeak": null, "onPct": null, "offPct": null } Rules: - label = start month of billing period, 2-digit year (e.g. "Jun 25" for a Jun–Jul bill) - days = integer - kwh = integer - cost = number, 2 decimal places (electricity charges only, exclude budget billing adjustments) - rate = "Standard" or "Time-of-Day" - If Time-of-Day: fill in onPeak, offPeak, onPct, offPct — otherwise set all four to null - temp = average temp °F if shown on the bill, otherwise null
Then commit it — pick whichever fits where you are:
1
Copy the JSON entry Claude returns
2
From any device (phone, tablet, work PC): go to github.com/scammy37/jcpl-dashboard → data.json, click ✏️, paste the entry, click Commit changes

From your home PC: open C:\Claude\jcpl-dashboard\data.json, paste the entry, save, then run git add data.json && git commit -m "Add [Month] bill" && git push
3
Dashboard updates within ~1 minute — reload the page

⚡ Option C — Run the Script Home PC only

Fastest when you're at your home PC. Get the JSON from Claude (Option B), then run add_bill.py — it updates data.json, commits, and pushes in one step.

1
Get the JSON entry from Claude using the Option B prompt
2
Open PowerShell in C:\Claude\jcpl-dashboard (right-click folder → Open in Terminal)
3
Run: python add_bill.py — paste the JSON and press Enter twice
4
Review the values, press Y to commit and push — done
💡 You can also pass the JSON as a direct argument:
python add_bill.py '{"label":"Jun 26","period":"Jun 04–Jul 03, 2026",...}'

🌐 Option D — Edit on GitHub Directly Any device

Browser only — no Claude needed if you already have the JSON or want to type it in manually.

1
Go to github.com/scammy37/jcpl-dashboard → data.json
2
Click the ✏️ pencil icon (Edit this file)
3
Add a comma after the last }, paste your new entry before the closing ]
4
Click Commit changes — GitHub Pages redeploys automatically within ~1 minute

📋 data.json Entry Format

Each bill is one line. Your most recent entry for reference:

{"label":"May 26","period":"May 05–Jun 03, 2026","days":30,"kwh":1717,"cost":312.30,"temp":66,"rate":"Time-of-Day","onPeak":331,"offPeak":1386,"onPct":19.3,"offPct":80.7}
💡 temp: If your bill doesn't show average temperature, use the monthly average for Rockaway, NJ and fill it in manually.

💡 cost: Use electricity charges only — exclude any budget billing adjustments or credits.

💡 Standard rate months: Set onPeak, offPeak, onPct, and offPct all to null.
